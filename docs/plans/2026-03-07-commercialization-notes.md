# Book Power Commercialization Notes

> Early-stage thinking — seed for a proper strategy session (see task 12).

## What exists today

- A working pipeline: book → analysis → tool generation
- A reference implementation proving the concept: jtbd-knowledge (13 specialized tools from 2 JTBD books)
- A natural distribution channel: Harmonica hosts
- A clean IP model: platform provides infrastructure, hosts bring their own licensed content (YouTube/Dropbox model)

## Revenue angles

- **Publisher bundles** — authors/publishers sell "book + interactive tools" packages
- **Consultant pipeline** — consultants pay for the tool that turns their methodology into client-facing MCP servers or session templates
- **Harmonica integration** — hosts upload books → get custom facilitation templates. Harmonica charges for generation
- **Standalone SaaS** — book-power as its own product, independent of Harmonica

## Open-source vs. closed

| Layer | What it is | OSS? |
|-------|-----------|------|
| Generic framework | Tool types, analysis pipeline, basic 5-tool scaffolding | Already OSS (MIT) |
| Smart generator | Content-aware tool selection, methodology detection — turns 5-tool commodity into 13-tool product | **This is the real IP** |
| Generated output | Embedded book content, specialized tools | Private per instance (copyrighted content) |

**Open-core option:** Basic generator is OSS (as today), smart generator is paid. The moat is quality of output, not secrecy of infrastructure.

**Key decision point:** Task 11 (smart MCP generator) is where the open/closed decision matters. The current basic generator being OSS is fine either way.

## Why this could be big

- No one else is doing book → structured tools (MCP servers, session templates, slash commands)
- The methodology book market is massive and underserved digitally
- Natural network effect: more books processed → better generator → more books
- Clean IP separation means no legal risk for the platform
